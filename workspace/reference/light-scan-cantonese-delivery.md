## 目標

將 cron job「light-scan」完成後發給用戶的結果，改為**粵語**、簡潔的用戶面向格式，並統一版面。

- 希望對用戶只顯示友善、粵語、不暴露 session/system 簡潔細節的摘要。

## 未來期望格式（用戶面向）

**語言：** 粵語（繁體）

**範例模板：**

```

- 掃描設定有效（27分鐘前）

- 日誌：已記錄

```

### 範例（實際效果）

**情況一：冇觸發，一切正常**

```
📡 今日第20次掃描 完成 ✅

- 掃描設定有效（27分鐘前）
- 條件觸發：✓

- 日誌：已記錄
- {2026-03-01 20:06 HKT}
```

**情況二：有觸發，要留意**

```
📡 今日第8次掃描 完成 ✅

- 掃描設定有效（12分鐘前）
- 條件觸發：✗

- 日誌：已記錄
- {2026-03-01 20:21 HKT}
```

---

### 「掃描設定有效（X 分鐘前）」係咩意思？

**「掃描設定」** 指 light-scan 用緊嘅**設定**：例如觸發閾值（T1/T2/T3/T4）、要監察嘅條件、邊啲嘢算「觸發」等，即係成個掃描邏輯背後嘅參數。

**「有效（X 分鐘前）」** 即係：呢套設定喺 **X 分鐘前** 被系統檢查過，確認冇壞、冇過期、可以照用；到今次掃描完成，仍然係用緊同一套設定。  
所以呢行係話你知：**掃描用緊嘅設定，幾耐前仲被驗證過**。數字越大只代表「驗證係更早之前做」，唔代表而家失效。

### 「條件觸發：✓ / ✗」（剔與交叉）

用 **條件觸發：** 加 剔／交叉 就夠：

- **條件觸發：✓** — 今次掃描冇任何條件被觸發，唔使做嘢，一切正常。
- **條件觸發：✗** — 今次有條件被觸發，有嘢要你跟進／留意。

✓（剔）= 冇觸發；✗（交叉）= 有觸發。

### 欄位對應（供實作參考）

| 用戶顯示 | 對應內部/邏輯 |
|---------|----------------|
| 今日第 N 次掃描 | `LIGHT_SCAN_COUNT`（或重置後的計數邏輯） |
| 掃描設定有效（X 分鐘前） | CONFIG 有效性與時間（掃描用緊嘅設定幾耐前被驗證過） |
| 條件觸發：✓ / ✗ | Triggers: NONE → ✓；有觸發 → ✗ |
| 日誌：已記錄 | Log appended |
| 時間戳 | 例如 2026-03-01 20:06 HKT |

### 不向用戶顯示的內容（保持內部）

- sessionId、cron job 名稱、runtime、tokens、prompt/cache
- 原始 system message 或 log 路徑
- 不要逐字複製 system message
