## Talk16 — 成個系統講緊咩

你開咗間店，賣一樣嘢（例如預測合約）。你要決定兩件事：
1. **我覺得呢樣嘢值幾錢？**（保留價格）
2. **我應該掛幾高賣、幾低買？**（最優報價寬度）

呢個框架就係用數學幫你做呢兩個決定，同時管理三樣風險：
- 你手上揸太多/太少貨（庫存風險）
- 市場突然大跳（跳躍風險）
- 有人用內幕消息食你（毒性風險）

---

## 第 1 步：保留價格（Micro-Pricing Agent）

### 公式
```
r(s, q, t) = s − qγσ²(T − t)
```

### 逐個拆

| 符號 | 意思 | 類比 |
|---|---|---|
| **s** | 市場中間價（mid-price） | 街市大家叫嘅價 |
| **q** | 你手上揸幾多貨（庫存） | +3 = 你買多咗 3 張，-2 = 你賣多咗 2 張 |
| **γ** (gamma) | 風險厭惡度 | 越大 = 越膽小，越想快啲清貨 |
| **σ²** (sigma²) | 價格波動嘅方差 | 市場越癲，呢個數越大 |
| **T − t** | 離結算仲有幾耐 | T = 到期時間，t = 而家 |

### 核心邏輯

**你嘅「心理價」= 市場價 − 庫存壓力**

庫存壓力 = `q × γ × σ² × (T−t)`

四種情況：

| 你嘅處境 | q 值 | 調整方向 | 原因 |
|---|---|---|---|
| 買多咗（長倉） | q > 0 | 心理價 **低過** 市場價 | 想快啲賣出去，寧願平啲放 |
| 賣多咗（短倉） | q < 0 | 心理價 **高過** 市場價 | 想快啲買返，寧願貴啲接 |
| 冇倉位 | q = 0 | 心理價 **等於** 市場價 | 冇壓力，客觀定價 |
| 市場快到期 | T−t → 0 | 調整量 → 0 | 快完，風險細，唔使急 |

### 三個放大器

「急唔急」取決於三樣嘢，互相相乘：

1. **γ（你嘅膽量）** — 越驚越急走。職業做市商 γ 低（見慣風浪），散戶 γ 高
2. **σ²（市場瘋狂度）** — BTC 跌 10% 嗰日 vs 橫盤日，壓力天差地遠
3. **T−t（剩餘時間）** — 仲有一個月 vs 仲有 5 分鐘，完全唔同嘅急迫感

> **一句總結**：保留價格 = 市場價 − （我嘅倉位 × 我嘅恐懼 × 市場嘅瘋狂 × 剩餘時間）

## 第 2 步：搵 Edge（Statistical Edge Detector）

### 公式
```
Edge = |r(s, q, t) − s|
```

即係：你嘅心理價同市場價差幾遠。

- Edge 大 = 你覺得市場定錯價，有得賺
- Edge 細 = 市場價合理，唔值得冒險

**框架要求 Edge > 3 basis points（0.03%）先出手。**

### 點解簡化成 |qγσ²(T−t)|

因為 `r(s,q,t) − s = −qγσ²(T−t)`，取絕對值就係：
```
Edge = |q| × γ × σ² × (T − t)
```

含義：你揸越多貨、越怕風險、市場越癲、時間越長 → Edge 越大 → 越有動機交易。

## 第 3 步：最優報價（Inventory-Aware Sizing Agent）

### 公式
```
δ^as(q) = (1/κ) × ln(1 + κ/γ) + (γ/κ)(Q − q) × f(γ, σ, κ, A)
δ^be(q) = (1/κ) × ln(1 + κ/γ) + (γ/κ)(Q + q) × f(γ, σ, κ, A)
```

### 拆解

| 符號 | 意思 |
|---|---|
| **δ^as** | Ask spread — 你嘅賣價同中間價差幾遠 |
| **δ^be** | Bid spread — 你嘅買價同中間價差幾遠 |
| **κ** (kappa) | 訂單到達強度 — 你掛嘅價有幾大機會被人食（spread 越窄越易成交）|
| **Q** | 最大允許庫存（硬限制）|
| **A** | 訂單到達率 |
| **f(γ,σ,κ,A)** | 調整函數（綜合所有參數）|

### 核心邏輯

兩部分：

**第一部分 `(1/κ)ln(1+κ/γ)`** — 基礎 spread，同庫存無關
- κ 大（好多人交易）→ 你可以掛窄啲，因為容易成交
- γ 大（你好怕）→ 你要掛闊啲，保護自己

**第二部分 `(γ/κ)(Q ∓ q)`** — 庫存調整
- Ask 用 `(Q − q)`：你已經揸好多貨（q 大）→ Q−q 細 → ask 收窄 → 更易賣出
- Bid 用 `(Q + q)`：你已經揸好多貨（q 大）→ Q+q 大 → bid 拉闊 → 更唔想再買

**效果**：當你倉位偏向一邊，系統自動傾斜報價去「糾正」倉位。

## 第 4 步：毒性偵測（VPIN — Risk Mitigation）

### 公式
```
VPIN = Σ(τ=1 to n) |V_τ^S − V_τ^B| / (n × V)
```

### 拆解

| 符號 | 意思 |
|---|---|
| **V_τ^S** | 第 τ 個時段嘅賣方成交量 |
| **V_τ^B** | 第 τ 個時段嘅買方成交量 |
| **n** | 時段數量 |
| **V** | 每個時段嘅平均成交量 |

### 核心邏輯

VPIN = 買賣力量幾唔平衡。

- VPIN 低 = 買賣差唔多 = 正常市場 = 安全做市
- VPIN 高 = 一面倒嘅買/賣 = 可能有人知道啲你唔知嘅嘢（內幕、大新聞）= 危險

**VPIN > 閾值 → 暫停交易。** 唔好做有毒流動性嘅對手。

### 點解重要

做市商最怕「adverse selection」— 即係有人用資訊優勢食你嘅 quote。你掛咗買盤，佢知道價格會跌就狂賣俾你。VPIN 就係偵測呢種模式。

## 第 5 步：執行 + 跳躍預測（Jump-Diffusion Model）

### 公式
```
dx_t = μ(x_t)dt + σ_b dW_t + J_t dN_t
```

### 拆解

| 部分 | 意思 | 類比 |
|---|---|---|
| **μ(x_t)dt** | 漂移項 — 價格嘅平均趨勢 | 河水嘅穩定流向 |
| **σ_b dW_t** | 擴散項 — 正常隨機波動 | 河面嘅小波浪 |
| **J_t dN_t** | 跳躍項 — 突然嘅大跳 | 石頭扔落水嘅大浪花 |

### 同普通模型嘅分別

普通 Black-Scholes 只有前兩項（漂移 + 波動）。但預測市場經常出現突發跳躍（例如選舉結果公佈、大新聞），所以加咗第三項 `J_t dN_t`。

- `N_t` = Poisson process — 跳躍幾時發生（隨機但有平均頻率）
- `J_t` = 跳躍幅度 — 跳幾多

**用途**：調整 limit order 嘅掛單距離同 time-in-force。預計有大跳 → 掛遠啲 / 快啲撤單。

## 進階模型（圖片第二頁）

以下五個模型係框架嘅延伸，難度較高，逐個簡講。

### 5. 非線性價格衝擊下嘅最優清倉

```
E₀ = sup_ν E[ q_T(s_T − k∫₀ᵀ q_t^α dt) + ∫₀ᵀ ν_t(q_t δ_t^{a*} − s_t) dt ]
```

**Talk16**：你要清一大堆貨，但你賣得越多，價格跌得越快。呢條公式搵「最唔傷嘅賣法」。

- `q_t^α` = 非線性衝擊（α > 1 時，大單嘅衝擊唔止係按比例放大，而係指數級放大）
- 經典 Almgren-Chriss 模型假設衝擊係線性嘅 → 呢度話佢太樂觀
- 實際含義：大倉位要分好多次慢慢清，唔好一次 dump

### 6. 流量調整高斯過程預測器

```
K(x, x') = σ_f² exp(−½(x−x')ᵀ L⁻¹(x−x')) + κ² ∫₀ᵛ (I_τˢ − I_τᴮ) dτ
```

**Talk16**：用機器學習（高斯過程）預測價格，但加咗一項 — 買賣訂單嘅不平衡。

- 第一項 = 標準 GP kernel（價格歷史嘅模式）
- 第二項 = 累積訂單不平衡（有人一直狂買 → 價格可能就嚟升）
- **問題**：κ 參數要實時校準，而且市場結構會變（non-stationary），所以 GP 嘅假設可能唔成立

### 7. 訂單簿熵 + 資訊衰減

```
H(t) = −Σ(i=1→B) p_i(t) ln p_i(t) − Σ(i=1→A) p_i(t) ln p_i(t)
```

**Talk16**：用 Shannon 熵量度訂單簿嘅「混亂程度」。

- 熵高 = 訂單分散喺好多價位 = 正常
- 熵突然暴跌 = 訂單集中 / 消失 = 有人撤走 = 可能就嚟大跳
- 用途：熵跌 → 提前收窄掛單或暫停做市

### 8. 強化學習做市效用函數

```
U(Q) = −e^{−γQ}  ⟹  Q_{t+1} = Q_t + a_t − (a_t × γ⁻¹ δ_t^{a*} + drift)
```

**Talk16**：用 RL 嚟決定最優庫存。

- `U(Q) = −e^{−γQ}` = CARA 效用函數（怕風險嘅程度同財富多少無關）
- Agent 嘅 action `a_t` = 買幾多 / 賣幾多
- 同第 1 步分析解嘅分別：RL 可以學到非線性嘅最優策略，但計算成本高好多
- 框架問題：RL 學到嘅結果同 GLFT 分析解差幾遠？如果差唔多就唔值得用 RL

### 9. 分數階 GARCH 波動持續性

```
ln(s_t) ∼ ARIMA(p, d, q) - FIGARCH(p, δ, q)
```

**Talk16**：價格波動有「長記憶」— 今日嘅波動同一個月前嘅波動有關。

- GARCH = 波動率模型（今日波動 ≈ 昨日波動 × 衰減 + 新 shock）
- FIGARCH 加咗 fractional operator `(1−L)^d` — 容許衰減速度唔係指數級，而係幂律衰減（慢得多）
- `δ` = long-memory 參數（0 到 1）
- 問題：呢個 δ 喺唔同資產類別穩唔穩定？定係要逐個市場校準？

## 風控門檻（執行前必過）

圖中 TECHNICAL md structure 列出嘅 4 道門：

```
1. r(s,q,t) edge > 3 basis points     ← 唔夠 edge 唔做
2. VPIN < threshold_limit              ← 毒性太高唔做
3. Position size within GLFT [Q−q, Q+q] ← 超倉位限制唔做
4. Estimated Jump Probability < critical_mass ← 預計大跳唔做
```

全部過晒先可以出手。任何一道唔過 → 等。

## 模擬場景：BTC 預測合約做市

### 場景設定

你係一個 Polymarket 風格平台上嘅做市商，賣「BTC 31/3 結算時 > $95,000」嘅合約。

| 參數 | 值 | 解釋 |
|---|---|---|
| **s** | $0.62 | 合約中間價（市場覺得 62% 機率 BTC > 95K）|
| **q** | +5 | 你已經揸咗 5 張 YES 合約（之前買多咗）|
| **γ** | 0.1 | 中等風險厭惡（職業做市商水平）|
| **σ²** | 0.04 | 日波動率 20% → 方差 = 0.2² = 0.04 |
| **T − t** | 20 日 | 離結算仲有 20 日 |
| **κ** | 1.5 | 訂單到達強度（中等流動性）|
| **Q** | 10 | 最大允許庫存 |

### Step 1：計保留價格

```
r = s − qγσ²(T − t)
r = 0.62 − 5 × 0.1 × 0.04 × 20
r = 0.62 − 0.40
r = 0.22
```

**你嘅心理價 = $0.22**，遠低過市場嘅 $0.62。

含義：你已經買多咗 5 張，市場又癲（σ² = 0.04），仲有 20 日先結算 → 你非常想快啲賣出去。你覺得呢啲合約對你嚟講只值 $0.22（因為揸住有風險成本）。

### Step 2：計 Edge

```
Edge = |r − s| = |0.22 − 0.62| = 0.40 = 40 cents
```

40 cents >> 3 basis points → Edge 超大 ✓。但呢度 edge 大唔係因為「市場錯價」，而係因為你倉位壓力太大。

### Step 3：計最優報價

簡化計算（只取基礎 spread）：

```
基礎 spread = (1/κ) × ln(1 + κ/γ)
           = (1/1.5) × ln(1 + 1.5/0.1)
           = 0.667 × ln(16)
           = 0.667 × 2.773
           = 1.849

Ask 調整 = (γ/κ)(Q − q) = (0.1/1.5)(10 − 5) = 0.333
Bid 調整 = (γ/κ)(Q + q) = (0.1/1.5)(10 + 5) = 1.000
```

注意：呢個基礎 spread 值偏大係因為 γ 相對 κ 嘅比例。實際部署會根據 `f(γ,σ,κ,A)` 做縮放。

**效果方向**（重要嘅係相對關係）：
- **Ask spread 窄** → 你更願意平啲賣（因為你買多咗，想清貨）
- **Bid spread 闊** → 你唔想再買（已經揸夠）

### Step 4：VPIN 檢查

假設最近 50 個時段：

```
VPIN = Σ|V_S − V_B| / (n × V)

觀察到：大部分時段買賣差唔多，但最後 3 個時段突然一面倒賣
VPIN = 0.35
閾值 = 0.50
```

VPIN 0.35 < 0.50 → 安全 ✓。但已經偏高，要留意。

### Step 5：跳躍檢查

```
Jump Probability（由 logit model 估計）= 12%
Critical Mass = 25%
```

12% < 25% → 安全 ✓。

### 最終決策

| 門檻 | 值 | 通過？ |
|---|---|---|
| Edge > 3bp | 40 cents | ✓ |
| VPIN < 0.50 | 0.35 | ✓ |
| Position ∈ [Q−q, Q+q] | 5 ∈ [5, 15] | ✓ |
| Jump Prob < 25% | 12% | ✓ |

**結果：執行。** 掛一個偏向賣出嘅報價（ask 窄、bid 闊），盡量減少庫存。

### 情景演變

**10 日後（T−t = 10）：**
```
假設你賣咗 3 張，q = 2
r = 0.62 − 2 × 0.1 × 0.04 × 10 = 0.62 − 0.08 = 0.54
```

庫存壓力大幅下降：心理價從 0.22 → 0.54，接近市場價。報價趨向對稱。

**1 日前（T−t = 1）：**
```
假設 q = 1
r = 0.62 − 1 × 0.1 × 0.04 × 1 = 0.62 − 0.004 = 0.616
```

快到期，調整量幾乎為零。你嘅心理價 ≈ 市場價。

### 呢個模擬教咗咩

1. **庫存 × 時間 = 雙重壓力**：q=5 + T−t=20 嘅壓力 >> q=1 + T−t=1
2. **保留價格係動態嘅**：唔係設一次就算，每秒都喺變
3. **報價傾斜 = 自動修正機制**：揸多咗就自動平啲賣、貴啲買，推動倉位回歸中性
4. **四道門缺一不可**：Edge 再大，VPIN 爆咗一樣唔做

## 同 AXC 嘅關係

呢個框架係 **prediction market 做市** 用嘅，同 AXC 而家做嘅 **方向性交易** 有根本分別：

| | GLFT 做市 | AXC 方向交易 |
|---|---|---|
| 目標 | 賺 bid-ask spread | 賺方向性 move |
| 庫存 | 盡量維持中性 | 主動揸倉 |
| 風險 | adverse selection | 方向判斷錯 |
| 頻率 | 極高（秒級） | 中等（4H） |
| 適用場景 | 流動性提供 | 趨勢/回歸交易 |

**可以借鑑嘅概念**：
- VPIN 毒性偵測 → 判斷市場係咪「有人知道啲嘢你唔知」→ 高 VPIN 暫停入場
- 跳躍風險模型 → 大事件前（Fed meeting、ETF decision）自動收窄倉位
- 庫存壓力思維 → AXC 已有嘅 position sizing 可以加入「現有倉位修正因子」
