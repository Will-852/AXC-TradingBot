OpenClaw

AI 加密貨幣交易系統 — 系統說明文件

01

OpenClaw 係咩？

一個 24 小時自動運行嘅 AI 助手，幫你監察加密貨幣市場、分析走勢、自動下單、Telegram 通知結果。你唔需要盯住電腦。

02

系統運作流程

掃描員 ──→ 分析員 ──→ 決策員 ──→ 交易員 ↑ ↓ 心跳監察 Telegram 通知你 新聞員 ──→ 情緒分析 ──→ 輔助判斷
  • 掃描員 — 每 3 分鐘掃描市場，搵大波動
  • 分析員 — 計算技術指標（BB、ATR、支撐阻力）
  • 決策員 — 綜合數據判斷入唔入場
  • 交易員 — 向交易所落盤、設止蝕止賺
  • 新聞員 — 抓取新聞、分析市場情緒

04

交易模式

模式觸發門檻風險適合
CONSERVATIVE3%低（1% risk, max 1 倉）市場平靜
BALANCED2.5%中（2% risk, max 2 倉）一般市況
AGGRESSIVE2%高（3% risk, max 3 倉）市場活躍
揀定一個模式後，最少跑 30 次交易先換。頻繁切換令數據冇參考價值。
05

風控機制

系統有多重自動保護，唔需要手動操作：

機制觸發條件行為
單筆熔斷單倉虧損 ≥ 25%即時強制平倉
日度熔斷當日虧損 ≥ 15%停止當日所有交易
連虧冷卻連輸 2 次暫停 30 分鐘
重度冷卻連輸 3 次暫停 2 小時
持倉上限3 天未平倉自動平倉（MAX_HOLD_HOURS=72）
資金費率資金費 > 50% 未實現盈虧強制平倉
低流動性成交量 < 50% 均值唔入場
極端費率資金費率 ≥ 0.2%唔入場
再入場縮減虧損後再入場倉位自動縮減 30%
如果你見到系統冇交易，最常見原因係連虧冷卻或者市場波動唔夠。呢啲都係正常保護機制。

換 API Key

OpenClaw 有兩個地方存 API key，兩邊都要改：

文件管邊個點改
openclaw.jsonGateway + AI agentsopenclaw config set
secrets/.envPython scripts（掃描、TG、新聞）nano 編輯

換 AI API Key（最常見）

複製# 第 1 步：改 Gateway（3 個 tier） openclaw config set models.providers.tier1.apiKey "sk-新key" openclaw config set models.providers.tier2.apiKey "sk-新key" openclaw config set models.providers.tier3.apiKey "sk-新key" # 第 2 步：改 Scripts nano ~/.openclaw/secrets/.env # 搵 PROXY_API_KEY=舊key → 改成新key # Ctrl+O 存檔，Ctrl+X 退出 # 第 3 步：重啟全部服務 launchctl stop ai.openclaw.scanner launchctl stop ai.openclaw.telegram launchctl stop ai.openclaw.gateway sleep 3 launchctl start ai.openclaw.scanner launchctl start ai.openclaw.telegram launchctl start ai.openclaw.gateway
兩邊嘅 key 必須一致，否則部分服務用舊 key 失敗。
其他 API Key 場景（Proxy 地址 / 交易所 / Telegram）

換 API Proxy 地址

複製# Gateway openclaw config set models.providers.tier1.baseUrl "https://新地址/v1" openclaw config set models.providers.tier2.baseUrl "https://新地址/v1" openclaw config set models.providers.tier3.baseUrl "https://新地址/v1" # Scripts nano ~/.openclaw/secrets/.env # 改 PROXY_BASE_URL # 重啟 launchctl stop ai.openclaw.scanner && launchctl start ai.openclaw.scanner

換交易所 API

複製# 只需改 secrets/.env（Gateway 唔直接用交易所 key） nano ~/.openclaw/secrets/.env # 改 ASTER_API_KEY + ASTER_API_SECRET # 重啟 launchctl stop ai.openclaw.scanner && launchctl start ai.openclaw.scanner

換 Telegram Bot

複製# 1. 同 @BotFather 拎新 token # 2. Gateway openclaw config set channels.telegram.botToken "新token" # 3. Scripts nano ~/.openclaw/secrets/.env # 改 TELEGRAM_BOT_TOKEN # 4. 重啟 launchctl stop ai.openclaw.telegram && launchctl start ai.openclaw.telegram

常見問題

系統冇交易，點解？
  • 市場波動唔夠大（低於觸發門檻）→ 正常，繼續等
  • 連虧冷卻中（2 次暫停 30min，3 次暫停 2hr）→ Telegram /health 查看
  • 日度熔斷（當日虧 ≥15%）→ 第二日自動恢復
  • 暫停模式 → Telegram /resume 恢復
  • 服務停咗 → launchctl list | grep openclaw 確認
Telegram 冇收到通知？
複製launchctl list | grep openclaw

確認 ai.openclaw.telegram 有 PID（第一欄有數字）。冇嘅話：

複製launchctl start ai.openclaw.telegram
儀表板開唔到？
複製cd ~/.openclaw && python3 scripts/dashboard.py &

然後瀏覽器開 http://localhost:5555

掃描器停咗？
複製# 1. 檢查狀態 launchctl list | grep scanner # 2. 檢查日誌 tail -20 ~/.openclaw/logs/scanner.log # 3. 如果有鎖死 rm ~/.openclaw/shared/scanner_runner.lock # 4. 重啟 launchctl stop ai.openclaw.scanner && launchctl start ai.openclaw.scanner
災難恢復（系統壞晒點算？）

完整恢復約 15 分鐘，7 個步驟：

  1. Clone repo：git clone https://github.com/Will-852/openclaw.git ~/.openclaw
  2. 還原 .env：從 iCloud 或備份 zip 複製 secrets/.env
  3. 安裝依賴：pip3 install numpy requests
  4. 重建 RAG：python3 scripts/memory_init.py
  5. 驗證：bash scripts/health_check.sh
  6. 還原 LaunchAgents：複製 plist 到 ~/Library/LaunchAgents/
  7. 還原 crontab：crontab -e 加回 03:00 備份

詳見 docs/setup/RECOVERY.md

點解叫「人體架構」？

OpenClaw 用人體器官做隱喻。每個部分對應一個器官：有「大腦」負責思考、有「記憶」學習經驗、有「肌肉」負責行動。

┌─────────┐ │ SOUL │ ← 大腦：性格、決策風格 └────┬────┘ ┌────┴────┐ │ MEMORY │ ← 海馬體：過去經驗 └────┬────┘ ┌────┴────┐ │ DNA │ ← 基因：策略規則 └────┬────┘ ┌───────┴───────┐ │ MUSCLES │ ← 肌肉：交易引擎 └───────┬───────┘ ┌────┴────┐ │ BLOOD │ ← 血液：實時數據 └────┬────┘ ┌────┴────┐ │ MIRROR │ ← 鏡子：回測對照 └─────────┘
11

六大 LAYERS

SOUL — 靈魂
agents/*/SOUL.md | 從不自動改

每個 AI agent 嘅性格設定。掃描員著重「發現異常」，交易員著重「風險控制」。

MEMORY — 記憶
workspace/MEMORY.md | 每日自動

記住每次交易經驗。用 RAG 向量搜索儲存，AI 決策時自動翻查。

DNA — 基因
config/params.py | 每週回顧

從交易經驗歸納出嘅規則。每週策略回顧分析近期交易，成功嘅寫入，失敗嘅淘汰。

MUSCLES — 肌肉
scripts/ | 穩定不變

實際執行交易嘅核心代碼 — 計算入場位、止蝕止賺，向交易所下單。

BLOOD — 血液
shared/ | 實時

來自交易所嘅實時價格數據，係決策嘅「養分」。

MIRROR — 鏡子
CLAUDE.md | 參考用

用歷史數據驗證策略有效性。回測結果影響 DNA 嘅更新方向。

1. BLOOD 送入實時數據 2. MUSCLES 計算技術指標（BB、ATR、S/R） 3. DNA 提供觸發門檻同風險參數 4. MEMORY 提供歷史經驗 5. SOUL 綜合以上做最終決策 6. MIRROR 事後覈查決策質素

交易策略

策略觸發條件SLTP (Min R:R)槓桿
RANGE（橫行）BB 觸碰 + RSI 反轉 + S/R + Stoch1.2x ATR2.3:18x
TREND（趨勢）4 指標對齊 + 回調買入 + 4H+1H 確認1.5x ATR3.0:17x
模式偵測（點樣分辨橫行 vs 趨勢）

系統用 4H K 線上嘅 5 個指標自動判斷市場模式：

  • RSI 位置（超買/超賣/中性）
  • MACD 方向（上升/下降/中性）
  • 成交量趨勢（放量/縮量）
  • 資金費率（正/負/極端）
  • BB 寬度（收窄 = RANGE，擴張 = TREND）

五票制：多數決定用 RANGE 定 TREND 策略。
