OpenClaw

一個自動幫你做加密貨幣交易嘅 AI 系統

OpenClaw 係咩？

想像你請咗一個助手，24小時不停地盯著加密貨幣市場。佢會：

  • 每隔幾分鐘掃描市場，睇下邊隻幣有異動
  • 分析價格走勢，判斷係咪值得入場
  • 自動下單入場，設好止蝕止賺
  • 用 Telegram 通知你每一個動作

你唔需要一直盯住電腦。OpenClaw 會幫你做。

系統係點運作嘅？

掃描員 ──→ 分析員 ──→ 決策員 ──→ 交易員 ↑ ↓ 心跳監察 Telegram 通知你 新聞員 ──→ 情緒分析 ──→ 輔助判斷

就好似一間公司咁：

  • 掃描員（Scanner）— 負責四圍「望」，睇邊隻幣有大波動
  • 分析員（Analyst）— 計算技術指標，判斷入場時機
  • 交易員（Trader）— 真正落盤買賣
  • 心跳（Heartbeat）— 確保系統冇死機
  • 新聞員（News Agent）— 睇新聞判斷市場情緒

左邊欄 LAYERS 係咩意思？

名稱中文意思更新頻率解釋
SOUL靈魂 / 人格從不自動每個 AI agent 嘅性格設定，決定佢點思考、點決策
MEMORY記憶每日系統記住之前嘅交易，學習成功同失敗嘅經驗
DNA基因 / 策略每週根據歷史交易歸納出嘅交易規則，越交越精準
MUSCLES肌肉 / 引擎穩定不變實際執行交易嘅核心代碼，止蝕止賺嘅計算邏輯
BLOOD血液 / 數據實時來自交易所嘅實時價格數據，系統嘅「養分」
MIRROR鏡子 / 對照參考用回測系統，用歷史數據驗證策略係咪有效

儀表板各部分說明

行動部署

顯示每隻幣當前狀態：

  • 綠點：已達觸發條件，系統可能入場
  • 黃點：接近觸發，仲差少少
  • 灰點：未達條件，繼續等
  • SL（Stop Loss）= 止蝕位，輸到呢個價就自動出場
  • TP（Take Profit）= 止賺位，賺到呢個價就自動出場
  • S / R（Support / Resistance）= 支撐位 / 阻力位

累積盈虧

顯示今日同總計嘅盈虧。曲線向上 = 賺緊錢，向下 = 蝕緊錢。

交易記錄

每一筆交易嘅入場價同出場價。
⚠️ 符號 = 測試記錄（唔係真實交易），可以忽略。

系統活動

系統最近做咗咩：心跳確認、模式切換、入場出場等等。

交易模式（打法）

系統有三種打法，用 Telegram 指令切換：

模式觸發門檻風險適合市況
CONSERVATIVE高（較難觸發）低市場平靜時
BALANCED中（2%波動）中一般市況
AGGRESSIVE低（較易觸發）高市場活躍時
建議：揀定一個模式後，最少跑 30 次交易先換。頻繁切換會令數據冇參考價值。

Telegram 指令大全

喺 Telegram 同 OpenClaw bot 對話，可以用以下指令：

查詢狀態

發送畀 Bot：
/status

顯示系統當前狀態、運行中嘅服務、最近交易。

切換打法

發送畀 Bot：
/mode BALANCED /mode CONSERVATIVE /mode AGGRESSIVE

查詢持倉

發送畀 Bot：
/positions

手動平倉

發送畀 Bot：
/close BTCUSDT

電腦指令（Terminal 用）

以下指令需要喺 Mac 嘅 Terminal 執行。唔熟悉指令嘅話，唔需要理呢部分，用 Telegram 就夠。

啟動所有服務

Terminal 入面，複製以下全部貼上：
bash ~/.openclaw/scripts/openclaw.sh start

停止所有服務

bash ~/.openclaw/scripts/openclaw.sh stop

查看服務狀態

bash ~/.openclaw/scripts/openclaw.sh status

健康檢查（確認系統正常）

bash ~/.openclaw/scripts/health_check.sh

期望看到：39+ pass / 0 fail。如果有 fail，截圖記錄。

備份系統

bash ~/.openclaw/scripts/backup_agent.sh

手動跑策略回顧

python3 ~/.openclaw/scripts/weekly_strategy_review.py

會分析歷史交易，輸出建議到 ai/STRATEGY.md。

查看儀表板

瀏覽器打開：
http://localhost:5555

常見問題

系統冇交易，點解？

  • 市場波動唔夠大（低於觸發門檻）→ 正常，繼續等
  • 熔斷器觸發（連輸太多次）→ 系統自動暫停保護
  • 服務停咗 → 跑 /status 或 health check

Telegram 冇收到通知？

launchctl list | grep openclaw

確認 ai.openclaw.telegram 有 PID（第一欄有數字）。

儀表板開唔到？

cd ~/.openclaw && python3 scripts/dashboard.py &

然後開 http://localhost:5555

想重置所有服務？

bash ~/.openclaw/scripts/openclaw.sh stop sleep 3 bash ~/.openclaw/scripts/openclaw.sh start
